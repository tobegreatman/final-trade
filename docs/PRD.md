## 1. 产品概述

### 1.1 产品定位

SmartStock 是一款面向 A 股个人投资者的**盘中实时交易辅助工具**，将《顶级股票交易系统》的规则化交易体系转化为可交互的 Web 应用。系统不提供自动交易，而是作为「纪律执行器」，帮助用户在盘中完成：判断大盘环境 → 策略选股 → 仓位计算 → 交易记录 → 复盘分析 的完整闭环。

### 1.2 目标用户

- 有一定交易经验但缺乏纪律性的 A 股个人投资者
- 希望将主观交易转化为系统化交易的散户
- 已阅读或正在学习结构化交易规则的进阶投资者

### 1.3 核心价值

| 维度 | 价值 |
|------|------|
| 纪律 | 强制执行交易前检查清单、止损规则、仓位上限 |
| 效率 | 一站式完成从大盘判断到下单决策的全流程 |
| 复盘 | 自动统计胜率/盈亏比，识别违规模式和连续亏损 |
| 学习 | 策略速查页面随时回顾规则，降低决策犹豫 |

### 1.4 技术架构

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | Vue 3.5 + Vite 6 + Vue Router 4 + Pinia 2 | SPA，Composition API |
| 后端 | Koa 2 + @koa/router + @koa/cors | 纯 API 代理层，无数据库 |
| 数据源 | 东方财富公开 API + 腾讯/新浪备用 | 通过后端代理避免 CORS，主用东方财富，失败自动切换 |
| 持久化 | localStorage | 用户数据全部本地存储 |
| 部署 | 静态部署 | `vite build` + Node 起后端即可 |

## 3. 功能模块详细需求

### 3.1 大盘状态 (Dashboard)

**路由**: `/`
**用途**: 盘前/盘中判断当前市场环境，决定是否开仓及仓位上限

#### 3.1.1 六维判据表（v4）

系统自动采集 6 个维度数据，逐条判断牛/震/熊信号：

| 维度 | 数据来源 | 牛市信号 | 熊市信号 |
|------|----------|----------|----------|
| MACD | 上证指数 MACD(12,26,9) | 金叉 + 零轴上方 + 柱状图放大 | 死叉 + 零轴下方 + 柱状图放大 |
| 涨跌家数 | 沪深京A股实时涨跌统计 | 上涨/下跌 > 2 | 下跌/上涨 > 2 |
| RSI | 上证指数 RSI(14) | RSI>60 且上行，或超卖回升 | RSI<40 且下行，或超买回落 |
| 融资余额 | 近 10 日融资余额趋势 | 5 日净增 > 0.5% | 5 日净减 > 0.5% |
| 量价配合 | 上证指数近 5 日量价关系 | 价涨量增 / 底部放量 | 缩量上涨 / 放量下跌 / 顶背离 |
| 北向资金 | 近 20 日北向资金成交额趋势 | 近 5 日均量 > 20 日均量 × 1.1 | 近 5 日均量 < 20 日均量 × 0.9 |

**后端接口**:
- `GET /api/market/indices` — 上证/深证/创业板指数 + 120 日 K 线 + MA20/60/120
- `GET /api/market/breadth` — 沪深京A股涨/跌/平家数（push2 ulist 优先，新浪批量回退）
- `GET /api/market/northbound` — 近 20 日北向资金成交额（datacenter-web RPT_MUTUAL_DEALAMT）
- `GET /api/market/margin` — 近 10 日融资余额数据

#### 3.1.2 综合判定

根据六维信号数量综合判定市场状态：

| 状态 | 条件 | 建议总仓位 | 推荐策略 |
|------|------|------------|----------|
| 牛市 | ≥ 4 个牛信号 | 80-100% | 趋势突破 |
| 偏多 | ≥ 3 个牛信号 | 50-70% | 回调买入 |
| 震荡 | 其他 | ≤50% | 回调买入 |
| 偏空 | ≥ 3 个熊信号 | 20-40% | 仅观望 |
| 熊市 | ≥ 4 个熊信号 | 0-20% | 空仓 |

需 ≥ 3 个信号方向一致才确认（`confirmed` 字段）。

#### 3.1.3 策略选股建议

根据六维判据结果自动匹配策略，**复用选股筛选页面的四层漏斗逻辑**生成选股条件，通过东方财富 xuangu API 实时查询候选股票。

**四层漏斗预设**（由 `getStrategyPreset()` 根据市场状态自动填充）：

| 市场状态 | 第一层排雷 | 第二层基本面 | 第三层景气度 | 第四层技术信号 |
|---------|-----------|------------|------------|-------------|
| 牛市 | 8 项全选（auto 项） | ROE>12%, 营收增速>10%, 利润增速>10%, PE 5-40, 市值≥30亿 | 无 | 趋势突破（MACD金叉+均线多头） |
| 偏多 | 8 项全选 | ROE>12%, 负债率<60%, PE 5-40, 市值≥50亿 | 无 | 回调买入（接近MA20+缩量） |
| 震荡 | 8 项全选 | ROE>12%, 负债率<60%, 现金流+, PE 5-30, 市值≥50亿 | 无 | 回调买入 |
| 偏空/熊市 | — | — | — | 不生成选股建议 |

排雷层固定选中所有 `auto: true` 的 8 项（非ST、非停牌、非北交所、非退市、审计意见、商誉<30%、质押<60%、上市>1年），与选股筛选页面一致。

**一句话选股**:
共用工具函数 `buildScreenerPrompt()` 生成自然语言条件（Dashboard 和 Screener 页面共用同一逻辑，避免 drift）。调用东方财富 xuangu API（`np-tjxg-b.eastmoney.com`）实时解析并返回匹配股票。用户可：
- 直接使用自动生成的条件查询
- 点击「编辑条件」自定义修改选股条件
- 点击「查询」提交自定义条件
- 点击「恢复默认」回到系统自动生成的条件

**提示词示例（偏多/回调买入）**:
> 非ST，非停牌，非北交所，非退市，审计意见为标准无保留意见，商誉占净资产比例小于30%，大股东质押比例小于60%，上市时间大于1年，ROE大于12%，资产负债率小于60%，市盈率5到40，流通市值大于50亿，股价大于60日均线，60日均线向上，股价接近20日均线，偏离不超过2%，5日均量小于20日均量的70%

**候选股卡片**展示：股票名称、流通市值、价格、涨跌幅、PE、PB、换手率、成交量。点击跳转股票池页面。

**后端接口**: `POST /api/stock/xuangu`（主用，自然语言选股）、`GET /api/stock/screen`（备用，本地两步筛选）

#### 3.1.4 做多窗口速判

3 项自动检测条件，满足即提示多头窗口开启：
1. 指数收盘价 > MA60
2. MA60 连续 3 天拐头向上
3. 涨跌比 > 2

#### 3.1.5 交易前检查清单

9 项必检项（来源 `PRE_TRADE_CHECKLIST`），以复选框形式呈现：
1. 大盘状态是否适合交易？（对照六维判定 + 决策树）
2. 该股是否通过排雷清单？（对照3.2）
3. 是否有明确买入信号？（对照4.1/4.2/4.3）
4. 止损位是否已设定？（ATR计算 + 条件单）
5. 盈亏比是否 ≥ 2:1？
6. 仓位是否在风险预算内？（5.3公式计算）
7. 是否已设条件单/提醒？（不依赖盘中判断）
8. 是否在冷静状态下决策？（不在开盘/收盘前10分钟冲动）

### 3.3 选股筛选 (Screener)

**路由**: `/screener`
**用途**: 四层漏斗式选股，生成东方财富 App/PC 可用的筛选条件

#### 3.3.1 第一层：排雷（12 项）

排除有重大风险的股票，8 项自动检测 + 4 项需人工查阅 F10：

| # | 条件 | 检测方式 |
|---|------|----------|
| 1 | 非 ST | 自动 |
| 2 | 非停牌 | 自动 |
| 3 | 非北交所 | 自动 |
| 4 | 非退市 | 自动 |
| 5 | 审计意见：标准无保留 | 自动 |
| 6 | 商誉/净资产 < 30% | 自动 |
| 7 | 大股东质押 < 60% | 自动 |
| 8 | 上市天数 > 250 天 | 自动 |
| 9 | 关联交易/营收 < 40% | F10 人工 |
| 10 | 无存贷双高 | F10 人工 |
| 11 | 董监高减持 < 5% | F10 人工 |
| 12 | 无重大诉讼/处罚 | F10 人工 |

**规则**: 至少勾选 5 项才能继续下一层。

#### 3.3.2 第二层：基本面（7 参数）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| ROE | ≥ 12% | 净资产收益率 |
| 营收增速 | ≥ 10% | 同比 |
| 利润增速 | ≥ 10% | 同比 |
| 资产负债率 | ≤ 60% | |
| 经营现金流 | 正值 | |
| PE 区间 | 5-40 | 排除极端估值 |
| 最小市值 | 30 亿 | 排除微盘股 |

#### 3.3.3 第三层：景气度（3 选 1）

| 选项 | 含义 |
|------|------|
| 机构增持 | 机构持股↑ + 北向持股↑ + 主力净流入 |
| 业绩超预期 | 净利润增速 > 30% + 营收增速 > 20% + 研报 ≥ 3 |
| 龙虎榜强势 | 近 3 日上榜 + 买方机构多 + 换手率 3-20% + 流通市值 > 30 亿 |

#### 3.3.4 第四层：技术信号（3 选 1）

| 策略 | 条件 |
|------|------|
| 趋势突破 | 放量突破 20 日高点 + MACD 金叉 + MA20↑ + MA60↑ |
| 回调买入 | 股价 > MA60 且 MA60↑ + 股价接近20日均线偏离不超过2% + 5日均量小于20日均量的70% |
| 底部右侧确认 | 从高点跌 > 40% + 缩量后放量 + RSI < 30 拐头 + MACD 金叉 |

#### 3.3.5 输出

根据勾选条件自动生成三部分输出（由共用工具函数 `buildScreenerPrompt()` 生成，与 Dashboard 策略选股共用同一逻辑）：

1. **一句话选股（移动端）** — 可直接复制到东方财富 App「一句话选股」，使用中文逗号分隔
2. **手动确认条件（PC端）** — 东方财富 NLP 不支持的条件，需人工确认（如龙虎榜机构席位、研报数量）
3. **PC 公式代码** — 可直接粘贴到东方财富 PC 端「公式编辑器」或通达信

### 3.4 仓位计算 (Position)

**路由**: `/position`
**用途**: 基于 ATR 的动态仓位计算、持仓管理、行业集中度监控

#### 3.4.1 ATR 动态仓位计算器

**输入**:
- 股票代码（带「获取 ATR」按钮，自动调用 K 线接口计算 ATR(14)）
- 买入价
- 目标价
- 策略选择（趋势突破 / 回调买入 / 底部右侧确认）

**自动计算**:

止损价 = 买入价 − N × ATR (N 由策略决定)

| 策略 | ATR 乘数 N | 回撤比例 | 建议持有期 |
|------|-----------|---------|-----------|
| 趋势突破 | 1.5 | 8% | 1-4 周 |
| 回调买入 | 2.0 | 15% | 2-8 周 |
| 底部右侧确认 | 3.0 | 20% | 1-6 月 |

仓位 = min(风险预算 / 止损比例, 总资金 × 25%)

其中:
- 风险预算 = 总资金 × 2%（单笔最大亏损）
- 25% 为单股仓位上限

盈亏比 = (目标价 − 买入价) / (买入价 − 止损价)

**展示**:
- 止损价、建议仓位（万元）、仓位占比
- 盈亏比数值（建议 ≥ 2:1）
- 止损/止盈可视化条形图

#### 3.4.2 行业集中度

- 按行业分组统计持仓占比
- 某行业 > 30% 时发出警告
- 可视化柱状图

#### 3.4.3 持仓管理表

| 字段 | 说明 |
|------|------|
| 代码 | 股票代码 |
| 行业 | 所属行业 |
| 买价 | 买入价格 |
| 止损价 | 初始止损 |
| 跟踪止盈 | 动态止损价（只能上移，不能下调） |
| 仓位 | 持仓金额 |
| 策略 | 策略名称 |
| 日期 | 开仓日期 |

**操作**:
- 添加持仓：从计算器结果直接添加
- 更新跟踪止盈：仅允许上调
- 删除持仓
- 总资金可配置（默认 100 万），持久化到 localStorage

#### 3.4.4 策略参数参考表

展示三种策略的完整参数对照：ATR 乘数、回撤比例、建议持有期、时间止损规则。

### 3.5 交易日志 (Journal)

#### 3.5.2 新增交易表单

**展开/收起**设计，包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| 股票 | 搜索选择 | 复用股票搜索 API |
| 买入价 | 数字 | |
| 数量 | 整数 | |
| 策略 | 下拉 | 趋势突破/回调买入/底部右侧确认 |
| 市场环境 | 下拉 | 牛市/偏多/震荡/偏空/熊市 |

| 交易前检查 | 9 项复选框 | 来源 PRE_TRADE_CHECKLIST |

**盈亏比提示**: 实时显示当前买价/止损/目标计算出的盈亏比，< 2:1 时红色警告。

**ATR 自动计算流程**:
1. 用户输入股票代码和买入价
2. 点击「获取 ATR」→ 调用 `GET /api/stock/:code/kline`
3. 取最近 15 根 K 线，计算 True Range 序列
4. ATR = SMA(TR, 14)
5. 自动填充止损价 = 买入价 − N × ATR
6. 自动填充目标价 = 买入价 + N × ATR × 2

## 4. 后端接口规范

### 4.1 基础信息

- 基地址: `http://localhost:3001/api`
- 开发代理: Vite 将 `/api` 代理到 `localhost:3001`
- 所有接口返回 JSON，格式: `{ ok: boolean, data?: any, error?: string }`
- 数据来源: 东方财富公开 API（`push2his.eastmoney.com`、`datacenter-web.eastmoney.com`、`np-tjxg-b.eastmoney.com`），后端添加 Referer 和 User-Agent 头

## 7. 交易规则体系（核心业务逻辑）

### 7.1 六维大盘判定算法（v4）

来源: `src/utils/marketJudge.js`

输入: 指数数据（K 线、MACD、RSI）、涨跌家数、融资余额、量价关系、北向资金

输出:
```typescript
{
  status: 'bull' | 'bull-lean' | 'neutral' | 'bear-lean' | 'bear'
  label: string       // 如「牛市」「偏多」
  tag: string         // CSS class
  maxPosition: string // 如 '80-100%'
  strategy: string    // 推荐策略
  strategyName: string // 推荐策略名称
  signals: object[]   // 各维度判断结果
  score: { bull, bear, neutral } // 信号计数
  confirmed: boolean  // ≥ 3 个同向信号
  longWindow: {       // 做多窗口速判
    conditions: object[]
    allPass: boolean
    message: string
  }
}
```

### 7.2 仓位计算公式

来源: `src/utils/position.js`

```
止损距离 = N × ATR(14)
止损价 = 买入价 − 止损距离
止损比例 = 止损距离 / 买入价
风险预算 = 总资金 × 2%
单笔仓位 = min(风险预算 / 止损比例, 总资金 × 25%)
盈亏比 = (目标价 − 买入价) / 止损距离
跟踪止盈初始价 = 买入价 × (1 − 回撤比例)
```

### 7.3 ATR 计算方法

```
True Range = max(当日最高 − 当日最低, |当日最高 − 昨收|, |当日最低 − 昨收|)
ATR(14) = SMA(TR, 14)  // 取最近 15 根 K 线，计算后 14 个 TR 的均值
```

## 8. 非功能性需求

### 8.1 性能

- 股票池行情每 `REFRESH_INTERVAL`（默认 10 秒）自动刷新，批量查询
- 服务端 API 缓存：涨跌家数 30 秒、其余接口 5 分钟，减少对东方财富的请求频率
- Dashboard 涨跌家数独立 30 秒刷新定时器，确保盘中数据实时性
- 东方财富 API 请求串行执行，避免并发触发限流
- 请求超时 8 秒自动中断，不会无限等待
- Sparkline/KlineChart 使用 Canvas 渲染，避免 DOM 开销
- K 线数据客户端缓存（klineCache），避免重复请求
- 搜索输入 300ms 防抖

### 8.2 容错

- API 请求失败时优先使用缓存数据，不进行降级处理
- 涨跌家数采用多级回退策略：push2 ulist（一次请求汇总沪深京A股）→ 新浪批量分页查询
- 东方财富 push2 系列域名（push2/push2his）不稳定时，自动切换到腾讯/新浪备用数据源（`server/fallback.js`）
- 北向资金使用 datacenter-web 的 RPT_MUTUAL_DEALAMT 报表，不依赖 push2 域名
- 选股功能：xuangu API 不可用时可回退到本地两步筛选（`/api/stock/screen`）
- 单个数据请求失败不影响其他数据的正常展示
- 图表加载中/加载失败/无数据三种状态明确区分
- 北向资金、融资余额、涨跌停数据持久化到 localStorage 作为本地缓存

### 8.3 兼容性

- 现代浏览器（Chrome / Edge / Firefox / Safari）
- 移动端响应式适配（768px 断点）
- Retina 屏幕 DPR 适配（Canvas）

### 8.4 数据安全

- 所有用户数据（自选、持仓、交易记录）仅存储在浏览器 localStorage
- 无服务端数据库，无用户注册/登录
- API 仅为东方财富公开数据的代理转发

### 8.5 可用性

- 无网络时：已缓存的自选列表和交易记录仍可查看（离线只读）
- API 失败时：静默降级，不阻塞页面操作
- 删除操作需二次确认
- 页面切换时自动启停对应定时器，不浪费资源

## 10. 页面流转

```
大盘判断(合理) → 策略选股 → 加入自选 → 股票池确认 → 仓位计算 → 记录交易
     ↓               ↓            ↓             ↓            ↓          ↓
  Dashboard      Dashboard    Watchlist     Watchlist    Position    Journal
                                                              绩效统计 + 复盘
```

**典型操作路径**:
1. 打开「大盘状态」→ 确认市场环境适合交易，查看策略选股建议
2. 点击候选股跳转「股票池」→ 查看走势和基本面
3. 或进入「选股筛选」→ 通过四层漏斗筛选候选股
4. 确认后进入「仓位计算」→ 计算 ATR、止损、仓位
5. 在「交易日志」新增交易记录
6. 平仓时填写复盘数据 → 在「绩效统计」和「复盘分析」中查看结果
7. 随时可查「策略速查」确认规则细节
