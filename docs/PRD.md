> 版本: 2.0
> 更新日期: 2026-05-12

## 1. 产品概述

### 1.1 产品定位

SmartStock 是一款面向 A 股个人投资者的**盘中实时交易辅助工具**，将《顶级股票交易系统》的规则化交易体系转化为可交互的 Web 应用。系统不提供自动交易，而是作为「纪律执行器」，帮助用户在盘中完成：判断大盘环境 → 选股筛选 → 仓位计算 → 交易记录 → 复盘分析 的完整闭环。

### 1.2 目标用户

- 有一定交易经验但缺乏纪律性的 A 股个人投资者
- 希望将主观交易转化为系统化交易的散户
- 已阅读或正在学习结构化交易规则的进阶投资者

### 1.3 核心价值

| 维度 | 价值 |
|------|------|
| 纪律 | 强制执行交易前检查清单、止损规则、仓位上限 |
| 效率 | 一站式完成从大盘判断到下单决策的全流程 |
| 复盘 | 自动统计胜率/利润因子，识别违规模式和连续亏损 |
| 学习 | 策略速查页面随时回顾规则，降低决策犹豫 |

### 1.4 技术架构

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | Vue 3.5 + Vite 6 + Vue Router 4 + Pinia 2 | SPA，Composition API |
| 后端 | Koa 2 + @koa/router + @koa/cors | 纯 API 代理层，无数据库 |
| 数据源 | 东方财富公开 API | 通过后端代理避免 CORS |
| 持久化 | localStorage | 用户数据全部本地存储 |
| 部署 | 静态部署 | `vite build` + Node 起后端即可 |

**服务端关键配置**:
- `dns.setDefaultResultOrder('ipv4first')` — 东方财富 IPv6 服务不稳定，强制 IPv4 优先
- `fetchJSON` 双重回退 — 先用 Node.js 内置 `fetch`（undici），失败后回退到原生 `https` 模块，解决 undici 部分接口 socket 兼容性问题

## 3. 功能模块详细需求

### 3.1 大盘状态 (Dashboard)

**路由**: `/`
**用途**: 盘前/盘中判断当前市场环境，决定是否开仓及仓位上限

#### 3.1.1 六维判据表

系统自动采集 6 个维度数据，逐条判断牛/震/熊信号：

| # | 维度 | 类型 | 数据来源 | 牛市信号 | 熊市信号 |
|---|------|------|----------|----------|----------|
| 1 | MACD | 动量 | DIF/DEA(12,26,9) + 零轴 + 柱状图 | 金叉+零轴上方+柱状图放大 | 死叉+零轴下方+柱状图放大 |
| 2 | 涨跌家数 | 市场广度 | 沪深两市实时涨跌统计 | 上涨/下跌 > 2 | 下跌/上涨 > 2 |
| 3 | RSI | 超买超卖 | RSI(14) 趋势和极值 | RSI>60上行 或 超卖回升 | RSI<40下行 或 超买回落 |
| 4 | 融资余额 | 杠杆资金 | 近 5 日融资余额变化趋势 | 5 日净增且增幅 > 0.5% | 5 日净减且降幅 > 0.5% |
| 5 | 量价配合 | 资金确认 | 价涨跌 + 量增减 + 离合检测 | 价涨量增 或 底部放量 | 顶背离 或 放量下跌 或 缩量上涨 |
| 6 | 北向资金 | 外资方向 | 近 5 日净流入数据 | 连续 ≥ 5 日净流入 | 连续 ≥ 5 日净流出 |

**数据缺失处理**: 北向资金全部为 0 时、融资余额数据不足 5 日时，判定为「数据不足」（中），不触发熊市信号。MACD、RSI、量价配合均从K线数据计算，无缺失风险。

**后端接口**:
- `GET /api/market/indices` — 上证/深证/创业板指数 + 120 日 K 线 + MA20/60/120（MACD/RSI/量价从此计算）
- `GET /api/market/breadth` — 沪深上涨/下跌/平盘家数
- `GET /api/market/northbound` — 近 5 日北向资金净流入
- `GET /api/market/margin` — 近 10 日融资余额数据
- `GET /api/market/limit-stats` — 当日涨停/跌停家数（展示用，不参与判定）

#### 3.1.2 综合判定

根据六维信号数量综合判定市场状态：

| 状态 | 条件 | 建议总仓位 | 推荐策略 |
|------|------|------------|----------|
| 牛市 | ≥ 4 个牛信号 | 80-100% | 趋势突破 |
| 偏多 | ≥ 3 个牛信号 | 50-70% | 回调买入 |
| 震荡 | 其他 | ≤ 50% | 回调买入 |
| 偏空 | ≥ 3 个熊信号 | 20-40% | 仅观望 |
| 熊市 | ≥ 4 个熊信号 | 0-20% | 空仓 |

需 ≥ 3 个信号方向一致才确认（`confirmed` 字段）。

#### 3.1.3 长窗口速判

3 项自动检测条件，满足即提示多头窗口开启：
1. 指数收盘价 > MA60
2. MA60 连续 3 天拐头向上
3. 涨跌比 > 2（上涨/下跌家数比）

#### 3.1.4 交易前检查清单

8 项必检项（来源 `PRE_TRADE_CHECKLIST`），以复选框形式呈现：
1. 大盘状态是否适合交易？（对照六维判定 + 决策树）
2. 该股是否通过排雷清单？（对照排雷项）
3. 是否有明确买入信号？（对照策略入场条件）
4. 止损位是否已设定？（ATR 计算 + 条件单）
5. 盈亏比是否 >= 2:1？
6. 仓位是否在风险预算内？（仓位公式计算）
7. 是否已设条件单/提醒？（不依赖盘中判断）
8. 是否在冷静状态下决策？（不在开盘/收盘前 10 分钟冲动）

- 渲染逻辑:
  - **1 日**: `<Sparkline>` 渲染当日分时数据，带昨收虚线，totalSlots=240
  - **1 周**: `<Sparkline>` 渲染 5 日分时合集（`intraday5d` 接口），前几天为 K 线 OHLC 合成数据，今天为真实分时
  - **1 月/3 月/1 年/5 年/全部**: `<KlineChart>` 渲染日 K 线切片
- 自动宽度，响应式 ResizeObserver
- 带面积渐变填充、高/低价格标注
- 参考线（昨收价）纳入 Y 轴范围计算，始终可见

**统计数据网格** (8 项):
- 日内最高、日内最低
- 区间最高（22 日）、区间最低（22 日）
- 市盈率 (PE)、市净率 (PB)
- 总市值、涨跌幅

**操作按钮**:
- 「计算仓位」→ 跳转 `/position?code=xxx`
- 「移除」→ 从自选列表删除

**后端接口**:
- `GET /api/stock/:code/kline` — 120 日 K 线
- `GET /api/stock/:code/kline5y` — ~5 年 K 线
- `GET /api/stock/:code/intraday` — 当日分时
- `GET /api/stock/:code/intraday5d` — 5 日分时合集（前几日 K 线合成 + 今日真实）
- `GET /api/stock/:code/basic` — PE/PB/市值等基本面

### 3.3 选股筛选 (Screener)

**路由**: `/screener`
**用途**: 四层漏斗式选股，生成东方财富 App/PC 可用的筛选条件

#### 3.3.1 第一层：排雷（10 项）

排除有重大风险的股票，6 项自动检测 + 4 项需人工查阅 F10：

| # | 条件 | 检测方式 |
|---|------|----------|
| 1 | 非 ST | 自动 |
| 2 | 非停牌 | 自动 |
| 3 | 审计意见：标准无保留 | 自动 |
| 4 | 商誉/净资产 < 30% | 自动 |
| 5 | 大股东质押 < 60% | 自动 |
| 6 | 上市天数 > 250 天 | 自动 |
| 7 | 关联交易/营收 < 40% | F10 人工 |
| 8 | 无存贷双高 | F10 人工 |
| 9 | 董监高减持 < 5% | F10 人工 |
| 10 | 无重大诉讼/处罚 | F10 人工 |

**规则**: 至少勾选 5 项才能继续下一层。

#### 3.3.2 第二层：基本面（7 参数）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| ROE | ≥ 12% | 净资产收益率 |
| 营收增速 | ≥ 10% | 同比 |
| 利润增速 | ≥ 10% | 同比 |
| 资产负债率 | ≤ 60% | |
| 经营现金流 | 正值 | |
| PE 区间 | 5-40 | 排除极端估值 |
| 最小市值 | 30 亿 | 排除微盘股 |

#### 3.3.3 第三层：景气度（3 选 1）

| 选项 | 含义 |
|------|------|
| 机构增持 | 机构持股↑ + 北向持股↑ + 主力净流入 |
| 业绩超预期 | 净利润增速 > 30% + 营收增速 > 20% + 研报 ≥ 3 |
| 龙虎榜强势 | 近 3 日上榜 + 买方机构多 + 换手率 3-20% + 流通市值 > 30 亿 |

#### 3.3.4 第四层：技术信号（3 选 1）

| 策略 | 条件 |
|------|------|
| 趋势突破 | 放量突破 20 日高点 + MACD 金叉 + MACD 柱 > 0 + MA20↑ + MA60↑ |
| 回调买入 | 股价 > MA60 且 MA60↑ + 缩量回调至 MA20 ± 2% + 量缩至 20 日最低 |
| 底部右侧确认 | 从高点跌 > 40% + 缩量后放量 + RSI < 30 拐头 + MACD 金叉 |

#### 3.3.5 输出

根据勾选条件自动生成：
1. **自然语言筛选语句** — 可直接复制到东方财富 App「一句话选股」
2. **PC 公式代码** — 可直接粘贴到东方财富 PC 端「公式编辑器」
3. **F10 提醒** — 标注仍需人工查阅的风险项

### 3.4 仓位计算 (Position)

**路由**: `/position`
**用途**: 基于 ATR 的动态仓位计算、持仓管理、行业集中度监控

#### 3.4.1 ATR 动态仓位计算器

**输入**:
- 股票代码（带「获取 ATR」按钮，自动调用 K 线接口计算 ATR(14)）
- 买入价
- 目标价
- 策略选择（趋势突破 / 回调买入 / 底部右侧确认）

**自动计算**:

止损价 = 买入价 − N × ATR (N 由策略决定)

| 策略 | ATR 乘数 N | 跟踪止盈 ATR | 回撤比例 | 建议持有期 | 单股仓位上限 |
|------|-----------|-------------|---------|-----------|------------|
| 趋势突破 | 1.5 | 2.0 | 8% | 1-4 周 | 25% |
| 回调买入 | 2.0 | 3.0 | 15% | 2-8 周 | 25% |
| 底部右侧确认 | 3.0 | 4.0 | 20% | 1-6 月 | 3% |

**波动率自适应止损**: 高波动（ATR/买价 > 5%）止损加 0.5N，低波动（ATR/买价 < 1.5%）止损减 0.5N。底部策略额外限制止损不超过买价 -8%（硬性上限）。

仓位 = min(风险预算 / 止损比例, 总资金 × 策略仓位上限)

其中:
- 风险预算 = 总资金 × 2%（单笔最大亏损）

盈亏比 = (目标价 − 买入价) / (买入价 − 止损价)

**展示**:
- 止损价、建议仓位（万元）、仓位占比
- 盈亏比数值（建议 ≥ 2:1）
- 止损/止盈可视化条形图

#### 3.4.2 回撤应对协议

| 级别 | 最大回撤 | 仓位缩放 |
|------|---------|---------|
| Level 0 | 5% | ×1.0（正常） |
| Level 1 | 10% | ×0.5（半仓） |
| Level 2 | 15% | ×0.2（轻仓） |
| Level 3 | 20% | ×0（空仓） |

#### 3.4.3 行业集中度

- 按行业分组统计持仓占比
- 某行业 > 30% 时发出警告
- 可视化柱状图

#### 3.4.4 持仓管理表

| 字段 | 说明 |
|------|------|
| 代码 | 股票代码 |
| 行业 | 所属行业 |
| 买价 | 买入价格 |
| 止损价 | 初始止损 |
| 跟踪止盈 | 动态止损价（只能上移，不能下调） |
| 仓位 | 持仓金额 |
| 策略 | 策略名称 |
| 日期 | 开仓日期 |

**操作**:
- 添加持仓：从计算器结果直接添加
- 更新跟踪止盈：仅允许上调（`newTrailingStop > current`）
- 删除持仓
- 总资金可配置（默认 100 万），持久化到 localStorage

#### 3.4.5 策略参数参考表

展示三种策略的完整参数对照：ATR 乘数、跟踪止盈 ATR、回撤比例、建议持有期、时间止损规则。

### 3.5 交易日志 (Journal)

**路由**: `/journal`
**用途**: 记录每笔交易的完整生命周期，自动统计绩效，辅助复盘改进

#### 3.5.1 页面结构

三个 Tab 页切换：

| Tab | 功能 |
|-----|------|
| 交易记录 | 新增/平仓/查看交易 |
| 绩效统计 | 胜率、利润因子、月度曲线 |
| 复盘分析 | 规则违反统计、系统警告 |

#### 3.5.2 新增交易表单

**展开/收起**设计，包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| 股票 | 搜索选择 | 复用股票搜索 API |
| 买入价 | 数字 | |
| 数量 | 整数 | |
| 策略 | 下拉 | 趋势突破/回调买入/底部右侧确认 |
| 市场环境 | 下拉 | 牛市/偏多/震荡/偏空/熊市 |
| 行业 | 文本 | 如「半导体」 |
| ATR | 只读 | 点击「获取 ATR」按钮自动填充 |
| 止损价 | 数字 | 可自动计算也可手动输入 |
| 目标价 | 数字 | 可自动计算也可手动输入 |
| 交易前检查 | 8 项复选框 | 来源 PRE_TRADE_CHECKLIST |

**盈亏比提示**: 实时显示当前买价/止损/目标计算出的盈亏比，< 2:1 时红色警告。

**ATR 自动计算流程**:
1. 用户输入股票代码和买入价
2. 点击「获取 ATR」→ 调用 `GET /api/stock/:code/kline`
3. 取最近 15 根 K 线，计算 True Range 序列
4. ATR = SMA(TR, 14)
5. 自动填充止损价 = 买入价 − N × ATR
6. 自动填充目标价 = 买入价 + N × ATR × 2

#### 3.5.3 持仓列表

展示所有 `status === 'open'` 的交易：

### 7.2 仓位计算公式

来源: `src/utils/position.js`

```
True Range = max(当日最高 − 当日最低, |当日最高 − 昨收|, |当日最低 − 昨收|)
ATR(14) = SMA(TR, 14)

基础止损距离 = N × ATR(14)
波动率自适应: 高波动(+0.5N) / 低波动(-0.5N)
底部策略硬性上限: 止损价 ≥ 买价 × 0.92 (最多亏8%)

止损价 = max(买价 − 止损距离, 0.01)
风险预算 = 总资金 × 2%
单笔仓位 = min(风险预算 / 止损比例, 总资金 × 策略仓位上限)
盈亏比 = (目标价 − 买价) / 止损距离
跟踪止盈初始价 = 买价 − 回撤AtrN × ATR
跟踪止盈更新: 仅上移 (newTrailing > currentTrailing)
```

### 7.3 合成分时算法

来源: `server/index.js` — `syntheticIntraday()`

当真实分时 API 不可用时，从日 K 线 OHLC 合成 240 点分时数据：

1. 判断涨跌方向：`close >= open` 为涨
2. 设定插值路径：
   - 涨: `[open, low, high, close]` — 先探低再冲高
   - 跌: `[open, high, low, close]` — 先冲高再探低
3. 3 段线性插值（每段 80 点）: 0-80, 80-160, 160-240
4. 时间标签: 前 120 点映射到 09:30-11:30（`570+i` 分钟），后 120 点映射到 13:00-15:00（`780+(i-120)` 分钟）
5. `volume` 固定为 0，`avg === close`

- Retina 屏幕 DPR 适配（Canvas）

- 所有用户数据（自选、持仓、交易记录）仅存储在浏览器 localStorage

- API 仅为东方财富公开数据的代理转发

- 无网络时：已缓存的自选列表和交易记录仍可查看（离线只读）
- 北向资金 API 失败时：使用 localStorage 缓存的历史数据
- 指数分时 API 失败时：返回空数据，前端保留上次有效数据
- 个股分时 API 失败时：从 K 线 OHLC 合成分时数据兜底
- 删除操作需二次确认

## 10. 页面流转

```
大盘判断(合理) → 选股筛选 → 加入自选 → 股票池确认 → 仓位计算 → 记录交易
     ↓               ↓            ↓             ↓            ↓          ↓
  Dashboard      Screener    Watchlist     Watchlist    Position    Journal
                                                                    ↓
                                                              绩效统计 + 复盘
```

**典型操作路径**:
1. 打开「大盘状态」→ 确认市场环境适合交易
2. 进入「选股筛选」→ 通过四层漏斗筛选候选股
3. 将候选股加入「股票池」→ 查看走势和基本面
